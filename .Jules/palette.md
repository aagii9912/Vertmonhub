## 2025-05-15 - Standardizing Header Accessibility and Keyboard Navigation
**Learning:** In a culturally localized application (like Mongolian), it's crucial that hidden accessibility properties (`aria-label`) also match the default language of the application so that screen reader users receive a consistent and localized experience. Additionally, relying solely on standard browser outlines is often insufficient, so explicitly setting `focus-visible` classes significantly improves keyboard-only navigation visibility without disrupting mouse users.
**Action:** When working on interactive dropdowns or icon-only buttons in the future, I will always check for the presence of `aria-expanded` and `aria-haspopup` attributes, explicitly define `focus-visible` styles using Tailwind, and ensure that all aria-labels are translated appropriately.
